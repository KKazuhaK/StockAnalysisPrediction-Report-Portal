## 升级说明

v0.3.0 是数据库 schema 的版本边界。若数据库来自 v0.1.x，请务必先运行 v0.2.26，确认服务成功启动并完成 v0.2 schema 升级后，再安装 v0.3.0；不支持从 v0.1.x 直接跨版本升级到 v0.3.0。

已经运行 v0.2.0–v0.2.26 的数据库可以直接升级。v0.3.0 只会自动补充缺失的普通列和索引，不会自动转换、移动或删除现有业务数据。

报告去重身份现在不再包含 `kind`。如果历史数据中存在相同 `symbol/title + rdate + rtype`、但 `kind` 不同的重复报告，服务会拒绝启动并提示冲突；请先人工确认需要保留的记录，系统不会自动删除或合并。

## 主要变化

- 加固会话密钥、反向代理客户端地址识别、登录限流、密码策略和安全响应头。
- 新创建的 API Token 改为摘要存储；现有明文 Token 保持原位并继续兼容，便于按计划轮换。
- 加固应用桥接、Markdown/HTML 渲染和 PDF 导出边界，限制资源消耗与外部资源访问。
- 优化报告首页查询、批量任务写入、调度查询、Token 活跃时间写入和前端轮询。
- 报告接口统一使用数字 `id`，并由数据库约束报告去重身份。
- 新增报告查询复合索引，同时保留旧索引，升级过程不执行删除。
